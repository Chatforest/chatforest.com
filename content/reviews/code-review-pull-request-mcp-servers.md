**At a glance:** Code review is the workflow where AI assistants and MCP can have the highest single impact — review capacity is the bottleneck in modern software development, and AI-powered code analysis through MCP can scale review throughput without scaling headcount. [SonarQube MCP](https://github.com/SonarSource/sonarqube-mcp-server) (Kotlin, native in SonarQube Cloud, 2026.1 LTA) leads code quality integration with expanded actionable issue management. [Codacy MCP](https://github.com/codacy/codacy-mcp-server) (official) covers SAST, secrets, coverage, and PR analysis; new Guardrails product pairs MCP with CLI for agentic workflows. [Graphite GT MCP](https://graphite.com/docs/gt-mcp) (built into CLI, still beta) enables AI-driven stacked PR creation. [CodeRabbit](https://www.coderabbit.ai/) now has an official MCP server (`0ui-labs/coderabbit-mcp-server`) that can trigger reviews and generate reports — no longer just an MCP client. [Microsoft Azure DevOps MCP](https://github.com/microsoft/azure-devops-mcp) (official, public preview) fills the biggest enterprise gap with PR support and remote server in Microsoft Foundry. Community PR review servers connect LLMs to GitHub diffs for automated feedback. This is the **twentieth review in our [Developer Tools MCP category](/categories/developer-tools/)**.

*Refreshed May 2026: Azure DevOps MCP fills major enterprise gap; CodeRabbit launches official MCP server; SonarQube expands with actionable issue management; Codacy Guardrails pairs MCP+CLI; Qodo hands PR-Agent to community. Rating upgraded 3.5→4/5.*

The AI code assistant market ($4.7B in 2025, growing to $14.6B by 2033 at 15.3% CAGR) is crystallizing fast — GitHub Copilot, Claude Code, and Cursor have all crossed $1B ARR. But code generation speed has shifted the bottleneck downstream: maintainers, tech leads, and engineering managers now face more pull requests, more diffs, and more surface area to validate. The MCP ecosystem is responding with two approaches: **platform connectors** that bring existing code quality data into AI workflows (SonarQube, Codacy) and **review agents** that use LLMs to analyze diffs directly (community servers). A third category — **PR workflow tools** (Graphite GT MCP) — focuses not on review itself but on making AI-generated code reviewable. A fourth pattern is now emerging: **enterprise DevOps platform connectors** (Azure DevOps MCP) that integrate PR review workflows into the broader project management and pipeline context.

**Architecture note:** Code review MCP servers follow five patterns. **Code quality platform connectors** (SonarQube, Codacy) expose existing quality gates, vulnerability scans, and coverage data as MCP tools — the review intelligence lives in the platform, MCP provides the interface. **PR management servers** (gitlab-mr-mcp, GitHub MCP) give AI agents read/write access to pull/merge requests (diffs, comments, approvals) without performing the review itself. **LLM-powered review agents** (crazyrabbitLTC, praneybehl, Orcus2021) use external LLMs to analyze diffs and generate review comments — the MCP server orchestrates the review pipeline. **Stacked PR workflow tools** (Graphite GT MCP) help AI agents structure code changes into reviewable units. **Review platform bridges** (coderabbitai-mcp) let AI assistants interact with reviews from dedicated AI review platforms.

**SonarQube MCP** (SonarSource, Kotlin) is the most established code quality MCP server. It integrates with SonarQube Server or SonarQube Cloud to provide code quality and security analysis through the MCP interface. The server analyzes code snippets directly within agent context — AI assistants can check code against SonarQube rules without leaving their workflow. Supported across 11+ platforms: Claude Code, Cursor, Codex CLI, Gemini CLI, GitHub Copilot CLI, GitHub Copilot coding agent, Kiro, VS Code, Windsurf, Zed, and Antigravity. Requires JDK 21+. Available as Docker image (`mcp/sonarqube`).

In March 2026, SonarQube Cloud launched a **native embedded MCP server** — no installation or Docker required. The MCP server is built directly into SonarQube Cloud, removing the "Docker barrier" that was the main adoption obstacle for hosted integrations. SonarQube (17.7% SAST mindshare, 7.4M+ users) detects bugs, vulnerabilities, code smells, and security hotspots across 30+ languages.

**May 2026 updates — actionable issue management**: SonarQube MCP significantly expanded beyond read-only quality data. AI agents can now take direct actions on issues: **update issue status** (confirm, unconfirm, resolve, reopen with optional comments), **mark false positives** individually or in bulk, **mark issues as "won't fix"** in bulk operations across multiple findings, **assign and unassign** issues to specific users — all without leaving the AI assistant interface. **Dependency risk detection** via SonarQube Advanced Security is now available through MCP, surfacing vulnerable dependencies alongside code smell and security hotspot data. **SonarQube Server 2026.1 LTA** released (with patch releases 2026.1.1, 2026.1.2, and 2026.2.1 following) — LTA designation means enterprise teams can standardize on this version with extended support.

**Codacy MCP** (official, TypeScript, MIT) positions itself as "MCP-native" — designed for the agentic development paradigm. Tools span the full code quality spectrum: **repository management** (setup, organization listing, analysis retrieval), **code quality** (issue detection with severity/category/language filtering), **security analysis** (SAST, secrets scanning, dependency scanning, IaC scanning, CI/CD scanning, DAST, penetration testing), **file analysis** (coverage metrics, duplication detection, complexity analysis), **pull request tools** (PR listing, diff analysis, coverage reports), and **local analysis** (CLI-based quality checks). Supports Cursor, Windsurf, Claude Desktop, VS Code with Copilot. In supported IDEs, install directly from the Codacy extension. Codacy's MCP server extends beyond basic code scanning to include security categories (secrets, IaC, DAST) that SonarQube's MCP server doesn't expose directly.

**May 2026 — Codacy Guardrails**: Codacy launched [Guardrails](https://www.codacy.com/guardrails), a new product that pairs the Codacy MCP server with the Codacy CLI for agentic development workflows. Agents get full visibility into every issue with exact repo locations, can spot which critical code lacks tests and write them, and can configure Codacy directly (enable/disable rules in Coding Standards) without leaving the chat panel. Guardrails positions Codacy as an end-to-end code quality loop for AI agents: write → detect issues (MCP) → fix → verify coverage (CLI) — all in one session.

### Stacked PR Workflow (1 server)

| Server | Stars | Language | License | Tools | Key Feature |
|--------|-------|----------|---------|-------|-------------|
| [Graphite GT MCP](https://graphite.com/docs/gt-mcp) | — | Go | — | — | AI agents create stacked PRs from large diffs — built into CLI v1.6.7+ |

**Graphite GT MCP** (built into Graphite CLI v1.6.7+, Go, beta) solves a problem unique to AI-generated code: large, monolithic diffs that are hard to review. Instead of reviewing code, GT MCP helps AI agents **create stacked pull requests** — breaking large changes into smaller, focused, sequentially-dependent PRs that humans can review incrementally. The MCP server is built directly into the Graphite CLI (`gt` command). AI agents navigate the stack (`gt ls`, `gt down`, `gt up`), insert new changes (`gt create --insert -am <description>`), and manage the full stacking workflow.

This approach tackles the fundamental tension in AI-assisted development: AI can generate code faster than humans can review it. Stacked PRs restore human oversight by decomposing AI output into digestible units. Graphite's reviewer (Graphite Agent, launched 2026) complements GT MCP on the review side — while GT MCP structures code for review, Graphite Agent performs the actual review. Currently in beta; some workflows may not be fully supported. Requires Graphite CLI v1.6.7+ and a Graphite account.

**May 2026 — CodeRabbit official MCP server**: CodeRabbit launched `0ui-labs/coderabbit-mcp-server`, announced on [coderabbit.ai's blog](https://www.coderabbit.ai/blog/coderabbits-mcp-server-integration-code-reviews-that-see-the-whole-picture). This fundamentally changes CodeRabbit's MCP role — it is no longer only an MCP client. The server provides: **trigger code reviews** directly from AI assistants, **generate reports** for specific time periods, **PR analysis** with configurable focus (security, quality, etc.), **per-repo review configuration**, **review uncommitted changes** before pushing, **interactive commands** during the review process, and **health checks** for the CodeRabbit environment. Requires a CodeRabbit API key (`CODERABBIT_API_KEY`) and optionally a GitHub token. Node.js 18+ required. Works with Claude Desktop and Claude Code. This resolves the long-standing gap where developers couldn't trigger a CodeRabbit review through MCP — now they can.

**CodeRabbit** continues to operate as an **MCP client** for enriching its reviews with external context. It consumes context from connected MCP servers (Datadog, New Relic, SonarQube, Snyk, Grafana, Linear, Confluence, Slack, Notion, Context7) before starting a review — database schema changes get checked against data architecture documents, API implementations get verified against design patterns. 50+ native integrations. Setup takes under 10 minutes per MCP server connection. Now also available as an OpenAI Codex plugin (Codex shipped 90+ plugins in April 2026, CodeRabbit included).

**bradthebeeble/coderabbitai-mcp** (26 stars, TypeScript, MIT) is a community-built bridge for reading existing CodeRabbit reviews. Tools include: retrieve and analyze CodeRabbit reviews on PRs, extract detailed review information and line comments, mark comments as resolved with custom notes, and a `/coderabbit-review` slash command. Requires GitHub PAT with `repo` scope. Remains useful for reading review history even with the official server now available.

**kopfrechner/gitlab-mr-mcp** (86 stars, 25 forks, 122 commits, JavaScript, MIT) is the leading GitLab-focused MCP server for merge request management. Ten tools cover the full MR lifecycle: `get_projects` (list accessible projects), `list_open_merge_requests` (view open MRs), `get_merge_request_details` (comprehensive MR info), `get_merge_request_comments` (discussion and diff notes), `add_merge_request_comment` (general comments), `add_merge_request_diff_comment` (line-specific comments on diffs), `get_merge_request_diff` (access diffs), `get_issue_details` (issue context), `set_merge_request_title`, and `set_merge_request_description`. Requires Node.js 18+ and GitLab PAT with `api` scope. Docker deployment supported.

While our [GitLab MCP Server](/reviews/gitlab-mcp-server/) review covers the broader GitLab ecosystem (including the official built-in MCP and zereight/gitlab-mcp at 1.2k stars), gitlab-mr-mcp is notable here for its focus specifically on the merge request review workflow — its tools are tuned for reading diffs, posting comments, and managing the review conversation. Additional GitLab code review MCP servers exist from mehmetakinn, ffpy, and yu-ito (Photosynth), each targeting MR review automation.

**crazyrabbitLTC/mcp-code-review-server** (32 stars, 23 forks, JS/TypeScript, MIT) provides two tools: `analyze_repo` flattens a codebase using [Repomix](https://github.com/yamadashy/repomix), and `code_review` performs an LLM-powered review. The flattening step is key — Repomix converts a repository into a single file that fits within LLM context windows, enabling whole-project analysis rather than per-file review. Supports OpenAI, Anthropic, and Google models. Reviews are structured as JSON objects with issue categorization (security, performance, quality, maintainability), severity ratings, line numbers, and actionable recommendations. Automatic code chunking handles large projects. Includes CLI for standalone testing.

**praneybehl/code-review-mcp** (30 stars, 7 forks, TypeScript, MIT, npm: `@vibesnipe/code-review-mcp`) uses the Vercel AI SDK for multi-provider LLM support (Google Gemini, OpenAI, Anthropic). Analyzes git diffs for staged changes, HEAD comparisons, or branch differences. Customizable review context through task descriptions and project information. Markdown-formatted output. Works with Claude Code, Cursor, and Windsurf. Install via `npx` — no global setup required.

**Orcus2021/code-review-mcp-server** (6 stars, TypeScript, v1.5.1) is the most workflow-complete community server. Seven tools: `CodeReview` (git diff with review guidance), `GetLocalGitDiff` (raw diff), `CodeReviewWithGithubUrl` (fetch PR diffs), `AddPRSummaryComment` (post aggregate feedback), `AddPRLineComment` (inline comments on specific lines), `GetPRTemplate` (retrieve PR templates), `CreatePR` (generate PRs with auto-detected repo data). Notion integration lets teams define custom review guidelines; falls back to local markdown instructions. CI pipeline automation via n8n webhooks.

**Microsoft/azure-devops-mcp** (official, Microsoft) entered public preview on March 17, 2026 and is now available as a remote server inside Microsoft Foundry — no local setup required. The server provides AI agents with secure access to **work items, pull requests, builds, test plans, and documentation** from Azure DevOps organizations. April 2026 update added `repo_get_file_content`, `repo_list_directory`, and `repo_vote_pull_request` — giving agents full PR review participation. Personal access token (PAT) authentication is now supported alongside OAuth, simplifying integration with external tools and CI systems. Microsoft has indicated plans to eventually archive the local server repository and focus investment entirely on the remote version (to align with remote MCP GA). This closes the most significant enterprise gap in the code review MCP ecosystem — Azure DevOps serves millions of enterprise developers, and its absence from MCP coverage was a notable limitation called out in our March review.

### Notable Absences

**Qodo/PR-Agent** (10.5k stars, the most popular open-source AI PR reviewer) does not expose an MCP server. PR-Agent provides review, describe, improve, and ask tools via GitHub bot, but these aren't available as MCP tools. **May 2026 update**: Qodo announced it is [handing PR-Agent over to the community](https://www.qodo.ai/blog/qodo-is-handing-pr-agent-over-to-the-community/) — moving to a new GitHub organization (`The-PR-Agent`), returning to Apache 2.0 license, and forming a new governance committee with external maintainer Naor Peled (known for TypeORM). This community ownership transition could accelerate MCP integration, but nothing has been announced yet. Qodo Command (separate from PR-Agent) supports a `--mcp` flag, but this is not equivalent to a PR-Agent MCP server.

**Ellipsis** (AI code review + bug fixes) uses MCP servers internally as part of its architecture but doesn't provide a public MCP server for developers to integrate.

**CodeAnt AI, Panto AI, Bito, Sourcery** — all popular AI code review tools — have no MCP servers. They integrate via GitHub/GitLab apps and IDE extensions, not MCP.

**Gerrit** — No MCP server exists for the Gerrit code review system used by Android, Chromium, and other large open-source projects.

## Known Issues

1. **The biggest AI PR reviewer has no MCP server** — Qodo/PR-Agent (10.5k stars) is the most popular open-source AI code reviewer, but it doesn't expose MCP tools. It operates through GitHub webhooks, CLI commands, and IDE extensions. **May 2026 update**: Qodo handed PR-Agent to the community (new org `The-PR-Agent`, Apache 2.0, external governance). The community transition could eventually produce an MCP server, but nothing has been announced. This remains the largest single gap in code review MCP coverage.

2. **CodeRabbit official MCP server is new and unproven** — CodeRabbit launched `0ui-labs/coderabbit-mcp-server` (May 2026), resolving our previous Known Issue that CodeRabbit was MCP-client-only. However, the server is very new — adoption, tool stability, and feature completeness are untested at scale. The community `bradthebeeble/coderabbitai-mcp` (26 stars) remains the better-tested option for reading existing reviews.

3. **Community review servers use LLMs to review LLM-generated code** — Most community code review MCP servers (crazyrabbitLTC, praneybehl) work by sending diffs to another LLM for analysis. This creates a circular pattern when the code was itself AI-generated: AI writes code, then AI reviews AI's code through an MCP server that calls AI. The review adds latency and cost without necessarily adding the independent perspective that human review provides.

4. **SonarQube MCP requires JDK 21** — The SonarQube MCP server requires JDK 21+, which is a heavy dependency for teams not already running Java. SonarQube Cloud's native embedded MCP removes this requirement for Cloud customers. SonarQube Server users still need the JDK dependency. Codacy's TypeScript-based server is lighter to deploy.

5. **Graphite GT MCP is still in beta and Graphite-locked** — GT MCP only works with Graphite's stacking workflow. Teams using GitHub's native PR model, GitLab merge requests, or other branching strategies can't use it. The beta status (unchanged since July 2025 launch) means some workflows may break. No open-source alternative exists for AI-driven stacked PR creation.

6. **No code review analytics MCP** — No MCP server provides review metrics: time-to-review, review thoroughness, comment resolution rates, reviewer workload distribution, or review coverage. These analytics are critical for engineering managers but exist only in proprietary dashboards (Graphite, LinearB, Sleuth, Swarmia) with no MCP exposure.

7. **Community servers have low adoption** — The most-starred dedicated code review MCP server (crazyrabbitLTC) has only 32 stars. Compare this with the GitHub MCP server (28.2k+ stars) or even Chrome DevTools MCP (31k stars). Code review MCP adoption is still in very early stages despite being a high-impact use case.

8. **No Gerrit MCP server** — Gerrit, the code review system used by Android (Google), Chromium, Eclipse, LibreOffice, and other large projects, has no MCP server. Gerrit's review model (change-based, not PR-based) would benefit from MCP integration, but the community hasn't built one.

9. **Review and repository overlap is confusing** — GitHub's MCP server (28.2k stars) includes PR review tools (get_pull_request, create_review, list_reviews). GitLab's official MCP includes MR management. Azure DevOps MCP (now official) also exposes PR operations. These overlap with dedicated code review MCP servers, making it unclear when developers should use the platform server vs. a specialized review server. The specialized servers add LLM analysis or quality gate integration, but the boundary isn't always obvious.

10. **Azure DevOps MCP is still preview** — Microsoft's `azure-devops-mcp` is in public preview with plans to archive the local server in favor of the remote version (timed to remote server GA). Enterprise teams adopting it now should plan for the local→remote migration. The remote version in Microsoft Foundry removes setup complexity but adds a cloud dependency.

## Bottom Line

**Rating: 4 out of 5**

Code review and pull request MCP servers address the most critical bottleneck in modern software development — review capacity. **May 2026** brought two major gap closures: [Microsoft's Azure DevOps MCP](https://github.com/microsoft/azure-devops-mcp) (official, public preview, remote server in Microsoft Foundry) fills the biggest enterprise platform gap, and [CodeRabbit's official MCP server](https://github.com/0ui-labs/coderabbit-mcp-server) (`0ui-labs`, can trigger reviews) resolves the client-only limitation. [SonarQube MCP](https://github.com/SonarSource/sonarqube-mcp-server) (native in SonarQube Cloud, 2026.1 LTA) now offers actionable issue management — mark false positives, bulk actions, assign/unassign. [Codacy MCP](https://github.com/codacy/codacy-mcp-server) (MIT) added [Guardrails](https://www.codacy.com/guardrails), pairing MCP with CLI for end-to-end agentic code quality. [Graphite GT MCP](https://graphite.com/docs/gt-mcp) remains in beta for stacked PR creation. Community servers ([crazyrabbitLTC](https://github.com/crazyrabbitLTC/mcp-code-review-server), [praneybehl](https://github.com/praneybehl/code-review-mcp)) connect LLMs to diffs. [kopfrechner/gitlab-mr-mcp](https://github.com/kopfrechner/gitlab-mr-mcp) (86 stars, 10 tools) handles GitLab merge request management.

The **4/5 rating** (upgraded from 3.5/5) reflects: two significant gap closures in this refresh cycle (Azure DevOps official MCP, CodeRabbit official MCP server), SonarQube's expanded actionable issue management, Codacy's Guardrails agentic workflow product, genuine vendor commitment across the stack, and breadth of approach (quality platforms, PR management, stacked workflows, LLM review agents, enterprise DevOps). Points held back by: Qodo/PR-Agent (10.5k stars, the most popular open-source reviewer) still has no MCP server even after community handoff, CircularLLM-reviews-LLM patterns in community servers, low community adoption (32 stars for the top dedicated server), Graphite's persistent beta status, no review analytics MCP, and no Gerrit support.

**Who benefits from code review MCP servers today:**

- **SonarQube users** — Native Cloud MCP or self-hosted server with actionable issue management (mark false positives, bulk actions, assign issues) across 11+ AI platforms
- **Codacy users** — Full-spectrum code quality (SAST, secrets, coverage, PR analysis) + Guardrails for end-to-end agentic quality workflows
- **Graphite users** — GT MCP lets AI agents create stacked PRs, making AI-generated code reviewable by humans
- **Azure DevOps teams** — Official Microsoft MCP (public preview) with work items, PRs, builds, test plans; remote server in Microsoft Foundry
- **CodeRabbit users** — New official MCP server enables triggering reviews from AI assistants, generating reports, reviewing uncommitted changes
- **GitLab teams** — [gitlab-mr-mcp](https://github.com/kopfrechner/gitlab-mr-mcp) enables AI agents to manage MR workflows (86 stars, 10 tools)
- **Teams wanting LLM-powered review** — Community servers add AI analysis on top of existing diffs

**Who should wait:**

- **Qodo/PR-Agent users** — The most popular open-source reviewer (10.5k stars, now community-governed) has no MCP server yet
- **Gerrit users** — No MCP server exists for Gerrit's change-based review model
- **Teams needing review analytics** — No MCP server provides review metrics or workload data
- **Teams wanting stable Graphite GT MCP** — Still in beta since July 2025; some workflows may break
